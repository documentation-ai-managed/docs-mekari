## Overview

Connect Mekari to your existing tools to automate workflows, sync data, and embed content seamlessly. You can integrate with third-party apps, set up webhooks for real-time updates, embed external resources, or build custom solutions using the API. These integrations help you centralize your documentation without switching contexts.

<Callout kind="tip">
  Start with pre-built third-party connections for quick setup, then explore webhooks and API for advanced automation.
</Callout>

## Webhook Setup for Automation

Webhooks enable real-time events from Mekari, such as document updates or new pages. Receive payloads at your endpoint to trigger actions.

<Steps>
  <Step title="Create Webhook" icon="plus">
    In Mekari dashboard, go to `Settings > Webhooks > New Webhook`. Enter your endpoint URL, like `https://your-webhook-url.com/mekari`.
  </Step>
  <Step title="Select Events" icon="zap">
    Choose events: `page.created`, `page.updated`, `space.exported`.
  </Step>
  <Step title="Verify and Test" icon="check-circle">
    Mekari sends a test payload. Respond with HTTP 200 to confirm.
  </Step>
</Steps>

Here is an example payload for a `page.updated` event:

<CodeGroup tabs="JSON">
  ```json
  {
    "event": "page.updated",
    "spaceId": "spc_abc123",
    "pageId": "pg_xyz789",
    "timestamp": 1699123456,
    "userId": "usr_def456"
  }
  ```
</CodeGroup>

<Callout kind="alert">
  Secure your webhook endpoint with HMAC signatures using `YOUR_WEBHOOK_SECRET`.
</Callout>

## API Access for Custom Integrations

Build tailored integrations using Mekari's REST API at `https://api.example.com/v1`.

<Request tabs="cURL,JavaScript" show-lines="true">
  ```bash
  curl -X POST https://api.example.com/v1/pages \
    -H "Authorization: Bearer YOUR_API_KEY" \
    -H "Content-Type: application/json" \
    -d '{
      "title": "New Page",
      "content": "Hello from API!"
    }'
  ```
  ````javascript
  const response = await fetch('https://api.example.com/v1/pages', {
    method: 'POST',
    headers: {
      'Authorization': 'Bearer YOUR_API_KEY',
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      title: 'New Page',
      content: 'Hello from API!'
    })
  });
  ````
</Request>

<Response tabs="201,401">
  ```json
  {
    "id": "pg_abc123",
    "title": "New Page",
    "spaceId": "spc_xyz789",
    "createdAt": 1699123456
  }
  ```

  ```json
  {
    "error": "Unauthorized",
    "message": "Invalid API key"
  }
  ```
</Response>

<ParamField path="title" param-type="string" required="true">
  Page title, max 255 characters.
</ParamField>

<ParamField header="Authorization" param-type="string" required="true">
  Bearer token: `Bearer YOUR_API_KEY`.
</ParamField>
